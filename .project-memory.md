## 核心功能
1. **模块文章浏览**：首页 3 张模块卡片 → 点击进入文章列表 → 点击文章进入详情页
2. **学习进度追踪**：每篇文章右侧圆形复选框，点击标记已读（金色 ✓），进度存于浏览器 localStorage
3. **上一篇/下一篇导航**：文章详情页底部有导航按钮
4. **随机练习生成器**：模块三底部有 22 条练习的随机抽取功能
5. **深链接支持**：URL hash 对应模块或文章 ID，刷新不丢失当前页面
6. **访问计数器**：首页底部显示总访问次数，通过零依赖 Netlify Function（Node.js 内置 fs 模块存储于 /tmp）实现，完全免费、无需 npm 包

## 文件清单
- `index.html` — 唯一的主文件（包含 HTML + CSS + JS + 全部文章内容）
- `.git/` — Git 版本控制
- `.project-memory.md` — 本文件
- `netlify.toml` — Netlify 构建和函数配置
- `netlify/functions/counter.js` — 访问计数器 Serverless Function（零依赖）

## JS 关键函数（位于 index.html 底部 script 标签中）
- `articles` 对象 — 存储 39 篇文章的标题、内容、元数据（1166 行截止）
- `moduleOrder` — 每模块的文章 ID 顺序数组
- `showView(id)` — 切换 view 显示
- `navTo(target)` — 导航到模块列表或首页
- `openArticle(id)` — 打开文章详情页
- `navArticle(dir)` — 上一篇/下一篇
- `toggleCheck(id)` — 切换学习进度标记
- `loadChecks()` — 从 localStorage 恢复进度
- `randomPractice()` — 随机抽取练习
- `loadCounter()` — 页面加载时从 countapi.xyz 获取并显示访问次数
- `init()` — 页面初始化（恢复进度 + hash 路由匹配）

## 后续开发注意事项
- 所有内容在同一个 `index.html` 中，修改文章需在 JS 的 `articles` 对象中操作
- 新增文章需要同时在 HTML 的文章列表区域添加对应的 `<div class="article-row">` 条目
- 新增文章需在 `moduleOrder` 中添加对应的 ID
- 线上部署在 Netlify，修改本地文件后需要重新部署才能更新线上版本
